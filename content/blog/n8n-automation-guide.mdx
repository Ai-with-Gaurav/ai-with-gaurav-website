## What is n8n?

n8n is an open-source workflow automation platform that lets you connect different apps and services without writing code. Think of it as a visual programming tool where you drag and drop nodes to create powerful automations.

## Why n8n Over Zapier or Make?

While Zapier and Make are great tools, n8n offers several advantages:

- **Self-hosted** — Your data stays on your servers
- **No execution limits** — Run as many workflows as you need
- **Custom code nodes** — Write JavaScript or Python when you need it
- **AI integration** — Native support for OpenAI, Claude, and other LLMs
- **Cost-effective** — Free for self-hosted, much cheaper than alternatives at scale

## Getting Started

### Self-Hosting with Docker

```bash
docker run -d \
  --name n8n \
  -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  n8nio/n8n
```

Visit `http://localhost:5678` and you're ready to build.

## Real-World Automation Examples

### 1. Contact Form to CRM Pipeline

When someone submits a contact form on your website:

1. **Webhook trigger** receives the form data
2. **AI node** analyzes the message to categorize the lead
3. **CRM integration** creates a new contact with proper tags
4. **Email node** sends a personalized auto-response
5. **Slack notification** alerts the sales team

### 2. Blog Post to Social Media

When you publish a new blog post:

1. **RSS trigger** detects the new post
2. **AI node** generates platform-specific social media copy
3. **Twitter/LinkedIn nodes** schedule posts across platforms
4. **Analytics node** tracks engagement after posting

### 3. AI-Powered Email Classification

For incoming support emails:

1. **Email trigger** captures new emails
2. **AI node** classifies urgency and category
3. **Router node** directs to appropriate team
4. **Ticket system** creates a prioritized ticket
5. **Auto-response** sends immediate acknowledgment

## Integrating AI into Workflows

n8n has built-in AI nodes that make it easy to add intelligence:

```javascript
// In a Code node
const response = await $http.post(
  'https://api.openai.com/v1/chat/completions',
  {
    model: 'gpt-4',
    messages: [
      {
        role: 'system',
        content: 'Classify the following email as urgent, normal, or low priority.'
      },
      {
        role: 'user',
        content: $input.item.json.emailBody
      }
    ]
  },
  {
    headers: {
      'Authorization': `Bearer ${$credentials.openaiApi.apiKey}`
    }
  }
);

return { priority: response.data.choices[0].message.content };
```

## Best Practices

1. **Start simple** — Automate one painful process before building complex workflows
2. **Error handling** — Always add error nodes to catch failures gracefully
3. **Monitoring** — Set up alerts for failed executions
4. **Version control** — Export your workflows as JSON and store in Git
5. **Security** — Use environment variables for API keys, never hardcode them
